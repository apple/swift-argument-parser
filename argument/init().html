Initializer

init()

This initializer works around a quirk of property wrappers, where the compiler will not see no-argument initializers in extensions. Explicitly marking this initializer unavailable means that when Value conforms to ExpressibleByArgument, that overload will be selected instead.

Declaration

init()

Discussion

@Argument() var foo: String // Syntax without this initializer
@Argument var foo: String   // Syntax with this initializer
